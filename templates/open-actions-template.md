## Open Actions Rules (Non-Negotiable)

1. If data is missing → create an Open Action. Never assume or fabricate.
2. If there is a mismatch between the risk assessment and the actual machine → create an Open Action.
3. If any schematic is missing or unverified → create an Open Action.
4. If a safety recommendation from the RA is not implemented on the machine → create an Open Action.
5. All Open Actions must be reviewed and signed off by the customer before the manual is finalised.
6. Open Actions do not block manual delivery — they are documented, not ignored.

## Liability Disclaimer for Open Actions

All items listed in this Open Actions Register represent areas where
original technical data was unavailable or where a discrepancy was
identified between the risk assessment and the actual machine.

Professional judgment and industry best practices have been applied
in these areas. All assumptions are clearly marked in the manual.

The customer is responsible for reviewing and resolving all Open Actions
before the machine is placed into service. Neither Fortress Safety nor
Ravikumar assumes liability for injuries arising from unresolved items
